Input types

We've got several different ways to make special input types.

Labeled Input boxes

Labeled input boxes have a helper label in them. You need to surround the input element with a span, and give than span the generic input-labeled class, and the type specific input-something class.

<span class="input-labeled input-search">
	<input type="text">
</span>
<span class="input-labeled flat input-search">
	<input class="flat" type="text">
</span>

Buttons

Default buttons

Button styles can be applied to anything with the .btn class applied. However, typically you'll want to apply these to only <a> and <button> elements for the best rendering.

Button class="" Description
btn Standard gray button with gradient
btn btn-primary Provides extra visual weight and identifies the primary action in a set of buttons
btn btn-info Used as an alternative to the default styles
btn btn-success Indicates a successful or positive action
btn btn-warning Indicates caution should be taken with this action
btn btn-danger Indicates a dangerous or potentially negative action
btn btn-inverse Alternate dark gray button, not tied to a semantic action or use
btn btn-link Deemphasize a button by making it look like a link while maintaining button behavior

Cross browser compatibility

IE9 doesn't crop background gradients on rounded corners, so we remove it. Related, IE9 jankifies disabled button elements, rendering text gray with a nasty text-shadow that we cannot fix.

Disabled state

Make buttons look unclickable by fading them back 50%.

Anchor element

Add the .disabled class to <a> buttons.

Primary link Link 

<a href="#" class="btn btn-large btn-primary disabled">Primary link</a>
<a href="#" class="btn btn-large disabled">Link</a>

Heads up! We use .disabled as a utility class here, similar to the common .active class, so no prefix is required. Also, this class is only for aesthetic; you must use custom JavaScript to disable links here.

Button element

Add the disabled attribute to <button> buttons.

<button type="button" class="btn btn-large btn-primary disabled" disabled="disabled">Primary button</button>
<button type="button" class="btn btn-large" disabled>Button</button>

One class, multiple tags

Use the .btn class on an <a>, <button>, or <input> element.

Link 
<a class="btn" href="">Link</a>

<input class="btn" type="button" value="Input" />

As a best practice, try to match the element for your context to ensure matching cross-browser rendering. If you have an input, use an <input type="submit"> for your button.

Dropdowns

Dropdowns require bootstrap-dropdown.js. You don't need to call a plugin on each individual element because event delegation handles this (looks for the data-toggle="dropdown" attribute). See Bootstrap's docs for more info. Note that we do not use the btn-group class like Bootstrap does. We use dropdown instead.

Retina Images

High resolution images are defined via CSS. If the user doesn't have a high-res monitor, we'll just serve them the normal image, but if they are capable, we'll serve them an @2x image. You won't notice a difference if you don't have a high-res monitor. The examples below are a comparison of what a retina image looks like compared to a standard image.

Set up

It takes some effort to get the retina images to work:

  1. You have to create an @2x image; double the resolution of the non-retina version.
  2. You have to create a @2x/ directory in the same path as where the original image is at. Within that @2x/ there has to be a file with the same name as the original.

For instance:

$ ls
  @2x/  a.png  b.png  c.png

$ ls @2x/
  a.png  b.png  c.png

Single High Res (Retina) Image

The method we use to serve high-res images is to use a @media query to detect the pixel-ratio and if its high enough it will serve the @2x image and scale it down to give a crisp look for high-density monitors (e.g. MacBook Pro Retina). Use the at2x() mixin found in styles/_mixins.scss.

Standard
High Res
/* Non-retina way */
#weebly-1x {
	background: image-url('style-guide/single/weebly.png');
	height: 200px;
	width: 250px;
}

/* Retina way; generates both a retina and non-retina version */
/* usage: @include at2x(PATH_TO_IMAGE, IMAGE_NAME[, IMAGE_EXTENSION]) */

#weebly-2x {
	@include at2x('style-guide/single/', 'weebly');
	height: 200px;
	width: 250px;
}

Backgrid Extension - Virual Cell

An in-house extension to Backgrid, since Backgrid doesn't provide a way to show a cell without an associated Model attribute. With the Virual Cell, you give the column a Backbone.View which will be render with access to the cell's Backbone.Model.

Backgrid isn't extremely flexible, so the cell attribute for the columns takes the VirualCell directly.

require(['backbone-all', 'backgrid', 'components/backgrid/backgrid-virtual'], function(Backbone, Backgrid) {

	// Backgrid Virtual Cell Extension
	// ================================

	var View = Backbone.View.extend({
		events: {
			'click': 'remove',
		},
		template: '<span>Delete</span>',
		render: function() {
			this.$el.empty().append(this.template);
			return this;
		},
		remove: function() {
			this.model.destroy();
		}
	});

	// VirtualCell needs to be extended with a Backbone.View
	var Cell = Backgrid.Extension.VirtualCell.extend({
		view: View
	});

	var columns = [{
		name: 'name',
		label: 'Name',
		cell: 'string'
	}, {
		name: 'legs',
		label: 'Legs',
		cell: 'integer'
	}, {
		name: 'eyes',
		label: 'Eyes',
		cell: 'integer'
	}, {
		name: '', // Name isn't needed since its virtual
		label: 'Delete',
		editable: false,
		cell: Cell // Place extended VirtualCell here!
	}];

	// Local Collection
	var animals = new Collection([
		{ 'name': 'Dogs', 'eyes': 2, 'legs': 4 },
		{ 'name': 'Cats', 'eyes': 2, 'legs': 4 },
		{ 'name': 'Cows', 'eyes': 2, 'legs': 4 },

});

DialogRegion

DialogRegion is a Custom Marionette Region Type. The Dialog Custom region is special in that it is created by extending it. This is because it extends Backbone.Events, which allows us to manage all of the Dialog's events within the View without the need to add it to the Dialog itself.

Dialog also supports new "flat" styling class.

var ExampleView = Backbone.Marionette.ItemView.extend({
	template: function() {
		return "Lorem ipsum dolor sit amet,...";
	},

	// custom configuration used by the DialogRegion
	dialog: {
		dialogClass: 'awesome-class',
		title: 'Awesome Example',
		buttons: [
			{
				text: 'Stuff',
				className: 'btn-com pull-left'
			},
			{
				group: 'close-group pull-right',
				buttons: [
					{
						text: 'Close',
						className: 'btn-com btn-success closeDialog'
					}
				]
			}
		]
	},

	events: {
		'click .closeDialog': 'closeDialog'
	},

	// trigger an event listened to by the DialogRegion
	closeDialog: function() {
		this.trigger('dialog:close');
	}
});

Adding Buttons

The buttons array in your view creates buttons in a new .dialog-footer div attached to the bottom of the dialog. You can specify either a single button or a group of buttons like in the example. Buttons can have either a text or html attribute for the button label, and also a className attribute. Button Groups have a group attribute which is translated to a div with the classnames you add to group, and it's own array of buttons.

Adding the Dialog Region

Create an instance of the Dialog Region by referencing it. This will assume you have a #dialog-region element in your view for the dialog to attach to.

require(['components/regions/Dialog'], function(DialogRegion) {
	app.addRegions({
		dialogExample: DialogRegion
	});
});

Otherwise, specify which element your dialog will attach to by extending the region like so:

require(['components/regions/Dialog'], function(DialogRegion) {
	app.addRegions({
		dialogExample: DialogRegion.extend({
			el: '#components-guide-dialog'
		})
	});
});

Then you just show the region like any other one:

 app.dialogExample.show(new ExampleView());

Updating your View inside of a Dialog

Sometimes your data changes, and you need to refresh the contents of your view. This is no different with a Dialog region. You simply want to re-render the view inside of your region, by calling this.render() inside of your view.

// inside of your view's initialize
this.listenTo(this.model, 'change', function() {
	this.render();
});

The Dialog Region will pick up on this re-render and refresh the buttons as well. However you can refresh the buttons yourself at anytime by calling this method from your view:

this.trigger('dialog:update:buttons', this.dialog.buttons);

You must pass in a button settings array. This can be any array so long as it follows the pattern explained above.

Updating the Dialog Title

You can easily change the title of the active Dialog by calling this event, along with the title:

this.trigger('dialog:update:title', 'My New Title');

Backbone Marionette EmptyLoadingView

EmptyLoadingView was created to be used with a Backbone Marionette CollectionView's emptyView attribute. The EmptyLoadingView can be used without any options, but can be customized to have a different loadingState, failureState, and emptyState.

Options

Name Type Default Description
loadingState string|Backbone.View "Loading" The intial loading state message
failureState string|Backbone.View "An error has occured, please try again" The failure state message
emptyState string|Backbone.View "No items to display" The true empty state message

See example usage below:

define(['backbone-all', 'components/views/EmptyLoadingView'], function(Backbone, EmptyLoadingView) {

	// EmptyLoadingView can be used with CollectionView and CompositeView
	var ExampleView = Backbone.Marionette.CompositeView.extend({

		...

		/**
		 * EmptyLoadingView takes advantage of the CollectionView's
		 * emptyView attribute, initially `this.collection` should
		 * be empty so that the emptyView is shown
		 */
		emptyView: EmptyLoadingView.extend({
			loadingState: "Loading text"
			failureState: Backbone.View.extend({
				template: _.template("I can be a Backbone View"),
				events: { 'click': function() { alert('') } }
			})
			//emptyState: "I come with default messages"
		}),

		/**
		 * In the initialization, prepare the collection to be empty and
		 * then fetch data for the collection.
		 */
		initialize: function() {
			var view = this;

			// First we must create an empty collection, this will force
			// our CollectionView to render the emptyView which will be
			// the EmptyLoadingView's emptyState (a loading animation)
			view.collection = new Backbone.Collection([]);

			// Fetch data to populate the collection
			Weebly.Editor.RPC.FetchExample.getAllPages(
				'parameter'
			).done(function(response) {

				// Handle success state
				if (response && response.success) {

					// Fetch was successful, repopulate our empty view.collection
					// with some real models, Marionette should automatically
					// rerender the items on reset, which will clear the initial
					// emptyView loading state
					view.collection.reset(response.data);

					// Check if the collection actually created any models
					// based off of the response, if length === 0, this is
					// a truly empty CollectionView, show EmptyLoadingView's
					// empty state message. To access the EmptyLoadingView, it
					// should be the CollectionView's first child.
					if (!view.collection.length) {
						view.children.first().showEmpty();
					}

				}
				// Handle error state
				else {
					// Trigger error method on EmptyView; to actually access
					// EmptyLoadingView, it should be the CollectionView's
					// first child, the `showFailure` method should be accessible
					view.children.first().showFailure();
				}

			}).fail(function() {
				// Trigger error method on EmptyView, same as above
				view.children.first().showFailure();
			});
		}
	});
});
